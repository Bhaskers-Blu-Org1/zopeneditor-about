# Setting language-specific tab stops and rulers

With VS Code, you can add tab stops and rulers in the editor depending on the language that you are using. You can customize the IBM Z Open Editor's settings based on your own personal perference or organization standards.

# Setting tab stops for COBOL

You can set tab stops for COBOL with IBM Z Open Editor in settings.json so that the cursor stops when you press the Tab key.

To configure COBOL tab stops, add zopeneditor.cobol.tabstops and provide an array of 0-based column numbers similar to the the following example to your user or workspace settings:

"zopeneditor.cobol.tabstops": [
    0,
    6,
    7,
    11,
    15,
    19,
    72
]

The default tab stops setting for COBOL is 0, 6, 7, 11.

# Setting language-specific rulers

You can set rulers for COBOL, PL/I and JCL, which are shown as grey lines, in the editor:

See the following example to set rulers for COBOL in settings.json:

"[cobol]": {
    "editor.rulers": [
        0,
        6,
        7,
        11,
        15,
        19,
        72
    ]
}

You can also set "[pl1]": {"editor.rulers": [array]} for PL/I and "[jcl]": {"editor.rulers": [array]} for JCL. array is an array of numbers representing the column numbers at which you want a ruler to appear.
